# 日历页面重构规划

## 1. 需求概述

用户希望新建一个日历页面来重构现有的日历样式，要求：
- 保留旧的页面代码（calendar.html, calendar.css）
- 风格简洁大方现代化
- 数据展示清晰明了

## 2. 现有代码分析

### 2.1 现有文件结构
```
CodeCal/
├── calendar.html      # 现有日历页面
├── calendar.css       # 现有日历样式
├── index.html         # 列表视图页面
├── index.css          # 主样式文件
├── platform-tag.css   # 平台标签样式
└── contests.json      # 比赛数据
```

### 2.2 现有技术栈
- FullCalendar 6.1.8
- Vue 3
- Tippy.js (tooltip)
- CSS Variables 主题切换

### 2.3 现有样式特点
- 复古卡片风格（box-shadow: 5px 5px 0px）
- 深色/浅色主题
- 平台标签颜色区分

## 3. 目标定义

### 3.1 新建页面
- 新文件名：`calendar-new.html`
- 新样式文件：`calendar-new.css`

### 3.2 设计目标
1. **简洁大方现代化** - 采用现代简约设计语言
2. **数据清晰** - 比赛信息展示直观易读
3. **交互友好** - 流畅的用户体验

## 4. 功能分解

### 4.1 核心功能
- [ ] 继承原有日历功能（展示近15天到未来一个月的比赛）
- [ ] 保持数据源不变（contests.json）
- [ ] 支持视图切换（周视图/月视图）
- [ ] 保留主题切换功能

### 4.2 UI 改进
- [ ] 现代化视觉设计
- [ ] 清晰的数据卡片
- [ ] 优化的移动端适配

## 5. 实施步骤

### 步骤 1: UI 设计（优先）
- 调用 UI-UX-Designer Agent
- 获取现代化日历界面设计方案

### 步骤 2: 创建新页面
- 创建 `calendar-new.html`
- 创建 `calendar-new.css`

### 步骤 3: 功能实现
- 集成 离线 Calendar.js
- 实现事件渲染
- 添加平台标签

### 步骤 4: 样式应用
- 应用现代化设计
- 确保数据清晰展示

### 步骤 5: 导航更新
- 在导航中添加新页面入口
- 保留旧页面访问

## 6. 验收标准

### 6.1 功能验收
- [ ] 新页面可正常加载
- [ ] 日历正确显示比赛数据
- [ ] 视图切换功能正常
- [ ] 点击事件可跳转

### 6.2 设计验收
- [ ] 风格简洁大方现代化
- [ ] 数据展示清晰明了
- [ ] 移动端适配良好
- [ ] 旧页面保持不变

### 6.3 质量验收
- [ ] 代码规范整洁
- [ ] 无控制台错误
- [ ] 加载性能良好
